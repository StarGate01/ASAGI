Detailed Description

See "Implementing Byte-Range Locks Using MPI One-Sided Communication" for details about this algorithm.

This class only works with at least 2 MPI processes.

This class will NOT lock MPIComm::mpiMutex!

long mpi::Mutex::acquire ( int  rank,
unsigned long  offset 
)
inline

Acquire a lock.

The caller has to make sure that the window is locked exclusively (e.g. with MPI_Win_lock) before/after the acquire call. This function will call MPI_Win_flush_local on the MPI window.

Parameters
rankThe rank where the lock is stored
offsetThe offset in the window (in window units) where the local is stored
Returns
A value < 0 if the mutex could be acquired. If a value >= 0 is returned the caller has to call wait(long) afterwards to finish the acquisition. In this case the mutex was locked.
